# AGENTS.md

このリポジトリでの作業ルール（事故防止の最小セット）:

1. ブランチは `main` のみを使う。別ブランチ運用をしない。
2. 実装前に「変更計画（対象ファイル・変更理由）」を提示し、承認を得る。
3. 変更は必要最小限のファイルに限定し、不要なリファクタを混ぜない。
4. `README.md` と `PROGRESS.md` は運用変更時に必ず同時確認する。
5. R2移行では段階移行を守る（いきなり削除しない）。
6. `public-data/songs.json` の削除は、`PROGRESS.md` の削除実行ゲートを全て満たした後のみ行う。
7. 障害時の復旧手順（ロールバック）を先に書いてから破壊的変更を行う。
